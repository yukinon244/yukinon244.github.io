gneH&Uy 2022春

今天考完。这门课是专门为大数据学院的同学开设的数据库课程，主讲老师是黄老师。

关于课程：该门课的教材选用的是王珊老师的《数据库系统概论》，上课内容是1-11章。大体内容是关系数据库、sql语句、数据库安全性、数据库完整性、关系数据理论、数据库设计、编程、关系查询处理和优化、数据库恢复技术、并发控制。

关于上课：黄老师第一次带课，但可以看出老师的课前准备做的很丰富，PPT做的很好，讲得比较的细致。只是由于本学期都是早八，可能大家听了很多概念后容易犯困。老师有时候会喊人起来回答问题，所以不要翘课。这门课的内容还是有点碎，知识点比较杂，需要注意的细节很多，黄老师也会在一些难以理解或容易犯错的地方提醒我们。

关于实验：本学期有两次实验。第一次实验是使用MySql去创建表实现40条sql语句功能，包含修改基本表、查询、索引、视图、触发器和开放题。实验难度不大，主要是让大家熟悉MySql的使用和sql语句。第二次实验是一次综合实验，这个实验的难度取决于你想要做到什么水平。内容大概是设计一个数据库系统，你可以选择使用C/S架构或者B/S架构，开发语言推荐如下：

python

C/S ：python + pyQt/Tkinter（python 下的图形界 面开发库）

B/S：python + Flask/Django（web 框架）

Java

C/S: java + Swing （java 图形界面库）

B/S: java + jsp （java 服务器动态页面语言）+ tomcat（severlet 容器，提供 jsp 服务）

Node

B/S: express框架 

需要自己设计前端和后端，以及将前后端连接起来，设计针对每个交互事件的响应函数。前端也可以采用Vue去实现。

同时在实验前，要提交ER图和需求分析，完成实验后会统一进行课程设计的展示，由老师助教评分。最后还要上交一份实验报告。数据库模式需要满足3NF。

在实现这个实验时，我还是遇到了不少困难，毕竟是第一次做，比如如何将前后端连接起来，以及学一些Javascript的语法。我在小组中主要负责后端数据库，以及前后端连接，主要采用flask。这个实验可以说是让我学到了很多，真正体会到设计一个数据库系统的不易。

关于考试：这学期的考试题量挺多，20分的选择，20分的填空。60分的大题。小题就不说了，大题大致是关系代数和sql语句的考察，ER图的设计和向关系模式的转化，函数依赖以及最小函数集，优化语法查询树，可冲突化串行调度，数据库设计（需求分析，概念结构设计，逻辑结构设计，物理结构设计）简答。总的来说考察还是比较全面的。小题考察的知识点比较碎，需要好好复习（或者背）。

关于给分：暂时未知。
