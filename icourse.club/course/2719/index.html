写在期末考试后，出总评之前

卫国：

这个老师喜欢课前提问：请大家用最抽象的话概括上节课学的东西。虽然鼓励开放的答案，但他本人却在找他期待的答案，即标准答案。这个提问纯属活跃气氛，不算分，考试也不是重点。

他对这门课的要求：

（1）学会独立思考（PPT上有错，半开卷别乱抄，可以参考陈佩青的书）。我觉得这就是托词，潜台词是“我也不知道，你先思考一下，把你思考的结果跟我讨论一下”

（2）学会原理性的东西。至于什么是原理性的东西呢？你去问他吧，我也没悟出来。

尹华锐：

上课喜欢吹垮垮。段子记住挺多。e.g.“苍蝇腿也是腿”

助教：

给1星的原因大部分在助教了。上午班的助教本科不是科大的，实验没做过，无法指导，问作业题也不回，在群里只催作业。

作业：

作业都是百度上能搜得到的

考试：

考试就非常阴间！没有往届的卷子，而且比作业题难一个数量级。老师的原话是：“你们要体谅老师啊，老师出一套好卷子真的不容易”。

分享卷子这件事嘛，也有弊端。如果考试本来就是课后难度，分享出来，就会一年比一年越卷，水涨船高，比如数学类课程的卷子。而DSP考试本来就比作业难一个数量级，所以我分享出来的目的就是破老师的题库。希望学弟学妹们也分享回忆版考卷，遍历老师的题库。

其他：

这门课只有一个老师上，本身就是一种垄断。垄断之后，怠于进步，教学质量下跌。实属信院的垃圾课之一。

整个一坨发烂发臭的狗屎。楼上@NULL2123 的评价十分到位了。学院的焯性大家懂得都懂，这门课基本上还要摆烂很多年，故而提供一个通关这门课的完整攻略：

1. 平时：不要听课！除非你有强大的屎海淘金能力。课上的东西听不懂，没用两者必占其一，最喜欢的就是吹捧所谓的“思想”，但是这种感性认识如果有了知识学生自己就能获得，不教知识只有感性认识就是典型的“思而不学则殆”。建议买一本清华出版社 程佩青 《数字信号处理教程》，照着看可以理解99%的知识点，剩下的1%要仔细从课件上看，注意和书上定义不同的和书上没有的，是考试重点。

2. 作业：有证据表明，题目近5年没有大的变化。从很多不同的教材上抄来的，网上有大部分答案。文末附上助教给的答案（这群懒狗很多是直接用工程软件算的，和手算有区别，不建议照抄）。

3. 实验：有证据表明，题目近十年完全没有变化。CSDN论坛有完整的三次报告，楼上有FFT的实现代码，检查实验完全没有查重（随助教变化这一点可能变化）。

4. 考试：今年和去年的试题已经在楼上有了，基本上题型固定。半开卷。文末是我的A4小抄供大家参考。最终的给分没有想象中的烂。

2022.1.10  本学期期末考试刚结束，在出分之前点评一下，以免总评影响判断

两位老师的上课风格其他同学已经说得很清楚了，总之非常玄乎，卫老师几乎不会给你做任何的数学推导。所以请上这门课的学弟学妹务必复习信号与系统知识，尤其是DTFT、Z变换的公式和性质之类。至于课前的提问与“交流”，只要不坐第一排和最后一排就好。

尹老师上课是这门痛苦课程中为数不多的欢乐时光（虽然但是，FFT的推导还是得认真听，因为实验要做）

虽然上课过程很折磨，但回过头来看，卫老师对于这门课的理解还是非常到位的。他非常乐意给同学们传递“思想”性的内容，想让大家真正以工科的思维去看待数字信号处理的过程，这在某种程度上来说确实是好事。但对于无法免受GPA枷锁的学生而言，从作业考试角度来说，卫老师的课程还是较为晦涩难懂。所以建议真正想学好这门课的同学，每周多花一点时间认认真真看一看教材，搞懂基本的数学公式与推导，再去听卫老师的课，会有很大收获的。（顺便一提，个人感觉王世一这本书还行，评论区其他同学说的程佩青那本教材更偏向于应试和做题技巧）

这学期的助教非常认真负责，作业批改得很细致而又合情合理，习题课讲得也很棒。

考试半开卷，可以带一张A4大小的Cheating Paper。但不得不疯狂吐槽的是他居然把我花了一下午+一晚上做的作弊纸给收上去了，本来还想留个纪念，气死。

function x = myFFT_PB16060919( x, N )
round = log2(N); % 绾ф暟
i = 1;
j = N/2;
X_even = x(1:i,1:j);
X_odd = x(1:i,j+1:j*2);
x = [X_even + X_odd; X_even-X_odd];
i = i * 2;
j = j / 2;
for k = 2:round
X_even = x(1:i,1:j);
X_odd = x(1:i,j+1:j*2);
temp = (0:i-1)';
temp = exp(-1j*2*pi*temp/(i*2));
temp = repmat(temp, 1, j);
X_odd = X_odd .* temp;
x = [X_even + X_odd; X_even-X_odd];
i = i * 2;
j = j / 2;
end
x = reshape(x, 1, N);
end

请问您今天要来点考试吗 2020秋

        数字信号处理是6系很重要的课程，需要有比较好的信号与系统基础。基本内容是：

         - 采样定理的深入理解与DFT，这部分的难点是 DFT 作为一个算法模型是如何近似 CFT 这么一个数学模型的，核心是采样定理的数字域和模拟域关联和转换，此外就是频谱混叠，频谱泄露，栅栏效应。这部分其实并不好理解，但却可以说是这个课最应该掌握的东西，建议认真多看几次这个（(12 封私信 / 15 条消息) 补零、频谱泄露、栅栏效应的关系？ - 知乎 (zhihu.com)）。

        用几句话说就是：1. CFT是时域连续谱+频域连续谱，时域采样会将时域离散化，频域连续谱发生复制（这个过程有混叠）；2. 由于我们不可能对无限长的序列分析（因为 DSP 就不能这样做，学了语音后会知道短时分析才有用），因此要对时域加窗处理进行序列选取（这个时候会在频域造成泄露现象，泄露的本质是窗函数频谱的卷积）3. 对频域离散化操作，同时时域会被复制（这个部分会导致栅栏效应）

         - FFT 各类算法。没啥好说的，FFT算法使用MATLAB复现注意一定避免使用for循环，通过合理安排系数位置可以做到不用for结构就实现FFT，速度能明显提高。可以逼自己禁用for语句，书写高效率代码。

         - IIR 滤波器设计法。后面的其实并没有什么大用......但考试还是重点。椭圆滤波和切比雪夫真的挺麻烦的，但说实话对以后学习没啥用。

         - FIR 滤波器设计。这里的窗函数一定需要多多理解，因为其他课程会有比较多的应用，理解窗函数是频谱泄露理解的关键。

        不能说老师讲的不行，不过6系烂课真的不少.......建议直接放弃课程教材，使用程佩青的教材。理解DFT真的多敲敲上面的代码。还有就是DFT的理解真的非常重要，高阶课程的很多地方，像语音处理，满屏都是DFT，不理解后面是根本不懂的。

        考试全是大题，前两个送分。答多少真的看运气......我1道完全没写，2道基本只做了一半，下考场直接情绪失控血崩。后来看成绩老师还是捞了一手的。总之不要因为这种不确定性影响心情才是重要的。

        再提醒一下，很多6系导师保研面试很喜欢问DSP和信号与系统的东西，基本集中在DFT，CFT，系统概念和性质这几点上。

       这门课课程难度困难主要体现在理解方面，尤其是课程开始时候对几个变换的理解以及变换域间的联系。讲课是分三个部分，先是卫老师前面的DFT，再是尹老师讲FFT和滤波器结构，最后卫老师再讲滤波器设计。第一部分我当时听得云里雾里，但到课程讲完才明白这部分重要性，这一部分一定要理解好，一定要认真听，对于公式推导别过分关注（考试是半开卷的），这也是第一次感觉专业课和学科基础课的区别；第二部分尹老师讲课类容差不多得了，讲的东西要么就是扯到实际的硬件结构，要么就是难以理解的，其实看看PPT也很容易的，理解部分不是很多，需要掌握思想的卫老师重开回来 是也会讲；第三个部分听得比较认真，卫老师对滤波器设计原理部分讲得非常好，认真听的话考试这部分不算难。
