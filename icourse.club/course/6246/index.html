liuly 2022春

非常建议计院大二下前来混分

CSAPP 内容比较大杂烩，其实挺适合做 CS 入门课，隔壁 pku 就是在大二上通修这门课

每个章节对应的内容及重要程度 https://icourse.club/course/6246/#review-7200 这个评课说的挺好的了，不再赘述

但上课实际覆盖到的内容比较少，可能只到第五章？（为什么不太确定是因为我也没怎么听过课）

这也导致如果想学到更多知识或者写更多实验的话肯定是得自己自学的

实验基本都有自动评测，难度确实不小（部分实验可能比大二下某 os 课程更难）

实验都在官网 http://csapp.cs.cmu.edu/3e/labs.html 下载，点 self-study handout, write up 是实验指导也需要下

实验确实比较老了，可能会碰到各种莫名其妙的环境问题。如果要做的话配一个 linux 环境是必须的，attack lab 新版本内核似乎跑不起来，建议使用 Vlab 或者从网上找一个别人配好的实验 docker 环境

对写了的实验点评一下：

  1. data lab 虽然简单，但确实是最无趣的一个实验，对 int 和 float 做各种二进制运算，网上资源比较全
  2. bomb lab 需要阅读大量的 x86 汇编，x86 汇编读起来没有 riscv 那么好懂，lab 是拆弹游戏形式，网上资源也比较全
  3. attack lab 缓冲区溢出攻击，也是游戏形式，几个 phase，分为两个阶段，第二部分 ROP 攻击难度大一点，网上也可能讲的不是很清楚。bomb 和 attack 都是汇编层面，也都比较有趣
  4. cache lab 第一部分缓存模拟器，第二部分矩阵乘法优化，算是比较有用的实验，个人感觉组原理论课出个这样的缓存模拟器实验显然比写一大堆莫名其妙的复读机题目要好得多
  5. malloc lab 实现 malloc 函数的分配算法，这个就到了系统层面，感觉应该是最难的一个实验，你的所有数据结构都必须在一片裸内存上自己维护，书上有 first fit 和 best fit 的代码，但如果想自己实现 segregated fit 等还是比较有难度的，而且很难 debug
  6. shell lab 这个 os 课应该会要求做，csapp 实验的特点是要求实现前后台进程组管理，代码框架已经给了，因此实验整体难度不大，对书上对应章节和几个系统调用的文档看仔细一点就很好做
  7. proxy lab 相当于 socket 编程结合 cache lab，之前做过 socket 相关的应该比较占便宜，比如大二下某 os 课程的多人聊天室
