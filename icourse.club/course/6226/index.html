printk 2017秋

帅老师是个很好的老师，没有大作业。

可能是教本科生不是非常特别深刻的语言底层细节理解，有一次看到一个同学问她 struct 可以作为参数而数组不可以的时候，问到压栈出栈、cdecl 传参数之类的细节的时候，她就又重复了一遍 struct 可以作参数而没有进行更深一步的讨论，而没有解释更深的实现机制等等。

其实科大的C语言教材确实算不上一本特别好的教材。教材里的代码排版参差不齐，有些地方并不严格按照C89、C99这些十几年前就已经确定的标准做事；有的时候还会看到一些小错误，比如 abs 函数应该在 而不是 。纵观全书，也没有特别提及头文件的重要性，以及封装这种编程思想；这方面推荐看看 aka 的《Linux C编程一站式学习》（Mirrors: https://akaedu.github.io/book/   GFDL许可证），我当初就是看的这个教材学习的C。

记得会有两次点名题，平时分除此之外由上机和作业情况决定。上机给助教检查的时候没有bug就算满分；作业同理。期末考试是纸质试卷，选择填空、程序阅读和程序编写题（现在只记得有链表了），看《上机指导书》（蓝皮，科大出版社出版，可以说是更烂的编码习惯了..）的模拟题就好。

平时不会的多翻翻 ISO C99 会很锻炼人，比如 static extern 的详解（Linkage、Duration和Initializer），比如指针（array[index] 和 index[array] 是等价的），比如文件操作（如果想用r+打开，读和写之间应该间隔一个fseek之类的操作），会对C语言有更深刻的理解。

期末考试有一个题目印象深刻，涉及到了有符号数符号扩展和无符号数扩展；有时间的时候会更新一下。
